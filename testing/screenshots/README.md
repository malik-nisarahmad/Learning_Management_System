# Screenshot Capture Guide

## How to Capture Screenshots for Test Report

### Using Windows (Snipping Tool)
1. Press `Windows + Shift + S`
2. Select the area to capture
3. Save to `testing/screenshots/` folder

### Required Screenshots

Capture and save the following screenshots:

| Filename | What to Capture |
|----------|----------------|
| `landing-page.png` | Full landing page with hero section |
| `login-page.png` | Login form (try invalid input to show error) |
| `register-page.png` | Registration form with validation |
| `dashboard.png` | Dashboard after login |
| `study-materials.png` | Study materials page with uploaded files |
| `ai-quiz.png` | Quiz page with generated questions |
| `quiz-results.png` | Quiz results showing score |
| `discussion-forum.png` | Discussion feed with posts and comments |
| `create-post.png` | Create post dialog |
| `threaded-comments.png` | Comments showing threaded replies |
| `chat-system.png` | Chat interface with conversations |
| `faculty-contact.png` | Faculty contact page |
| `ai-email-generation.png` | AI email dialog with generated content |
| `profile.png` | Profile management page |
| `events.png` | Events page |

### Tips for Good Screenshots
- Use full HD resolution (1920x1080)
- Show both success and error states
- Include console if showing API calls
- Capture loading states where relevant
